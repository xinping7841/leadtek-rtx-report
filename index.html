视频编解码服务器选型

针对视频服务器：不要只按 CUDA 核心或显存买卡。转码、直播、多路解码、云剪辑、代理文件生成这类场景，第一优先级是 NVENC/NVDEC 引擎数量和编解码格式支持，其次才是显存和 CUDA 性能。

关键结论：新采购优先 Blackwell。NVIDIA 官方 Video Encode/Decode Support Matrix 显示，Blackwell 专业卡的 H.265/HEVC 硬解覆盖 4:2:0 / 4:2:2 / 4:4:4 8/10/12-bit；Ada 与 Ampere 支持 HEVC 4:2:0 / 4:4:4 8/10/12-bit，但不支持 HEVC 4:2:2 硬解。做视频服务器时，4:2:2 源优先 Blackwell。

视频格式来源：编解码格式支持以 NVIDIA Video Encode/Decode Support Matrix 为准；H.265 分辨率上限参考 NVIDIA NVDEC Video Decoder API Programming Guide；单卡基础硬件参数仍以丽台官网产品页为准。

按你们用途的建议

  • 如果要新采购视频服务器：优先问 RTX PRO 5000 Blackwell、RTX PRO 4000 Blackwell、RTX PRO 4000 Blackwell SFF。它们在编解码能力、功耗、价格之间最容易做平衡。
  • 如果并发路数很多：不要默认买一张最贵卡。多张 RTX PRO 4000 Blackwell 或 RTX PRO 4000 Blackwell SFF 可能在 NVENC 数量、散热和冗余上更合适。
  • 如果素材有广播/专业相机 10-bit 4:2:2：优先 Blackwell；Ada/Ampere 不作为新主力。
  • 如果只是 H.264/H.265/AV1 4:2:0 转码：Ada 仍可用，库存价足够低时可以考虑；但新购仍建议优先 Blackwell。

视频解码排障与选型参数清单

现场判断口径：不要只看“支持 HEVC”。专业卡做视频解码时，要同时看编解码单元、Profile / 位深 / 色度格式、最大分辨率、实际 API 路径、输出 surface、多显卡显示环境、驱动版本和并发稳定性。

本次案例：测试视频为 HEVC Main、8-bit、4:2:0、7020x1800@25fps。现场 Ada 专业卡上 NVDEC / CUDA / D3D11VA 全片通过；DXVA2 在远程 SSH 会话里报 Failed to create Direct3D device，更像是 DXVA2/D3D9 设备创建或显示适配器选择问题，不是视频码流问题。

参数维度 需要记录/确认的项目 为什么重要 常见排查方法
解码器代际与数量 NVDEC / Video Decode Engine 代际、数量；必要时同时看 NVENC 数量。 多路解码吞吐主要受独立视频解码单元影响，不等同于 CUDA 核心数或显存容量。 查 NVIDIA Video Encode/Decode Support Matrix；运行 nvidia-smi dmon -s u 观察 decoder 占用。
Codec 细分能力 H.264、HEVC、AV1、VP9；HEVC Main / Main10 / Main12；8/10/12-bit；4:2:0 / 4:2:2 / 4:4:4。 “支持 HEVC”不代表所有 HEVC 都能硬解。Ada/Ampere 通常不支持 HEVC 4:2:2 硬解，Blackwell 才适合新建 4:2:2 工作流。 用 ffprobe 看 profile、pix_fmt、level；用官方矩阵核对色度和位深。
最大分辨率与吞吐 最大宽高、最大 luma samples、目标帧率、码率、并发路数。 单路 8K 可解不代表多路 4K/8K 稳定；非常规宽高也可能触发软件白名单或 API 兼容问题。 用真实素材跑全片测试；记录 width、height、coded_width、coded_height、fps。
实际 API 路径 NVDEC / CUVID、CUDA hwaccel、D3D11VA、DXVA2、Media Foundation、DirectShow。 同一张卡可能 NVDEC、CUDA、D3D11VA 都通过，但 DXVA2 失败。软件实际调用哪条路径决定现场结果。 分别跑 hevc_cuvid、-hwaccel cuda、-hwaccel d3d11va、-hwaccel dxva2 对照。
输出 surface 格式 NV12、P010、P016、YUV444、D3D11 surface、CUDA surface、是否回读 CPU 内存。 解码成功只是第一步。后续渲染、合成、转码是否零拷贝，会明显影响性能和延迟。 测试 -hwaccel_output_format d3d11 或 -hwaccel_output_format cuda，确认硬件帧路径能跑通。
多显卡与显示环境 独显、核显、远控虚拟显示器、HDMI 虚拟头、Windows 图形设置里的高性能 GPU 指定。 DXVA2/D3D9 特别容易受显示设备影响；远控软件和虚拟显示驱动可能导致 Direct3D 设备创建失败或选错 adapter。 用 Get-CimInstance Win32_VideoController 看所有显示适配器；本地显示器/虚拟头环境下复测。
驱动与软件版本 NVIDIA Studio / Production 驱动、WDDM、软件内置 FFmpeg/SDK/DirectShow/Media Foundation 版本。 旧驱动、旧 SDK 或软件硬编码白名单，会让硬件明明支持却不走硬解。 记录 nvidia-smi 驱动版本；升级驱动后复测；向软件方提供 FFmpeg 对照结果。
并发、延迟与稳定性 decoder utilization、显存占用、PCIe 传输、温度功耗、长时间播放、切片/GOP 跳转。 短样本能打开不代表生产稳定。多路、长时间、频繁 seek 更容易暴露吞吐和资源回收问题。 用全片和多路样本压测；同时看 nvidia-smi dmon、任务管理器 GPU Video Decode、应用日志。

HEVC 解码与并发关注点

HEVC 解码不是只看“支持/不支持”。视频服务器还要看 NVDEC 引擎数量、HEVC 位深、色度采样、目标分辨率、帧率、码率、是否同时做缩放/滤镜/AI 处理。本表每个型号的 H.265 硬解规格按 NVIDIA 官方专业卡 NVDEC 矩阵整理。

H.265 分辨率限制：本表这些 RTX A / RTX Ada / RTX PRO Blackwell 专业卡，H.265/HEVC 单路最大解码尺寸按 8192x8192 理解；同型号不同显存版本不会改变这个上限。实际能同时解几路 4K/8K，主要看 NVDEC 数量、码流复杂度、帧率、码率和后处理链路。

你们如果有 10-bit 或 12-bit 4:2:2 HEVC 源：优先 Blackwell。若主要是 8/10-bit 4:2:0 HEVC，Ada 仍然可用；Ampere 适合低价或存量。

注：RTX PRO 2000 Blackwell 的 NVENC/NVDEC 数量在丽台页面与 NVIDIA 产品页存在差异，本表按 NVIDIA 产品页列 1x/1x，采购前建议向代理确认。

采购判断

  • 新购优先级：同预算下优先看 Blackwell，尤其是 RTX PRO 4500 / 4000 / 2000；需要 48GB 以上显存再看 RTX PRO 5000 / 6000。
  • 大显存预算型：RTX 5880 Ada 现在比 RTX 6000 Ada 更容易形成价格优势；如果 RTX PRO 5000 48GB 报价接近 4 万以内，优先级通常更高。
  • 老款 Ampere：RTX A 系列适合存量项目、软件认证延续或低价库存。若报价接近 Ada/Blackwell，同价不建议优先采购。
  • 报价核验：采购前需要确认是否丽台行货、是否含税、质保年限、是否工包、是否附转接线/挡板、是否支持企业采购发票。
